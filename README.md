# Grade Clicker

**Grade Clicker** – это интерактивное приложение-кликер, разработанное в рамках лабораторной работы №12 по курсу «Разработка мобильных приложений».  
Пользователь набирает баллы, нажимая на изображение, а приложение визуально отображает прогресс: меняются фоновое изображение, цвет интерфейса и текстовое сообщение в зависимости от достигнутого порога баллов.

---

## Цель работы

Изучить жизненный цикл Activity (Activity Lifecycle) в Android и научиться правильно обрабатывать переходы между состояниями. В ходе выполнения лабораторной работы создано приложение **Grade Clicker**, которое демонстрирует:
- работу callback-методов жизненного цикла (`onCreate`, `onStart`, `onResume`, `onPause`, `onStop`, `onRestart`, `onDestroy`);
- сохранение состояния при изменении конфигурации устройства (поворот экрана) с помощью `rememberSaveable`;
- логирование событий жизненного цикла через класс `Log`.

## Контрольные вопросы

### 1. Какие три метода вызываются при запуске приложения?
При первом запуске приложения последовательно вызываются:
1. `onCreate()` – создание Activity;
2. `onStart()` – Activity становится видимой;
3. `onResume()` – Activity получает фокус и переходит в активное состояние.

### 2. Чем отличается onPause() от onStop()?
- `onPause()` вызывается, когда Activity теряет фокус, но остаётся частично видимой (например, при открытии диалогового окна или при вызове окна уведомлений).
- `onStop()` вызывается, когда Activity становится полностью невидимой (например, пользователь нажал кнопку «Домой» или переключился на другое приложение).

### 3. Когда вызывается onRestart()?
Метод `onRestart()` вызывается после того, как Activity была остановлена (вызван `onStop()`), но перед тем, как она снова станет видимой. Это происходит, когда пользователь возвращается к Activity, которая уже существовала в памяти (например, через экран недавних приложений).

### 4. Что происходит при повороте экрана?
При повороте экрана Android уничтожает текущую Activity (`onDestroy()`) и создаёт её заново (`onCreate()`), так как конфигурация устройства (ориентация) изменилась. Без специального сохранения данных (например, через `rememberSaveable`) все временные данные (баллы, количество кликов) теряются.

### 5. В чём разница между remember и rememberSaveable?
- `remember` сохраняет значение только при рекомпозиции (перерисовке UI), но теряет его при пересоздании Activity (например, при повороте экрана).
- `rememberSaveable` сохраняет значение как при рекомпозиции, так и при изменении конфигурации устройства (поворот, смена языка и т.д.), используя механизм `Bundle` (аналогично `onSaveInstanceState`).

### 6. Почему onCreate() должен вызывать super.onCreate()?
`super.onCreate()` вызывает реализацию метода в суперклассе (`AppCompatActivity`), где выполняются важные системные инициализации: связывание Activity с контекстом, создание оконного менеджера и другие внутренние настройки. Без этого вызова Activity не сможет корректно работать.

### 7. Для чего используется класс Log?
Класс `Log` предоставляет статические методы для записи сообщений в системный журнал Logcat. Это удобный инструмент для отладки: можно выводить сообщения с разными уровнями важности (`d()`, `i()`, `w()`, `e()`), чтобы отслеживать выполнение кода, состояние переменных и события жизненного цикла.

### 8. Какой метод вызывается только один раз за время жизни Activity?
Метод `onCreate()` вызывается только один раз за всё время существования экземпляра Activity. После уничтожения Activity и её последующего пересоздания `onCreate()` будет вызван снова.
